# Code Graph — лента алгоритмов рядом с кодом (VS Code, milestone 1)

Прототип переноса просмотрщика графов из браузера в расширение VS Code.
Показывает диаграммы «Алгоритм» (control-flow graph) **всех функций** файла C
вертикальной лентой в порядке исходника, **рядом с редактором**.

Код и граф намеренно **не синхронизированы непрерывно** (автопрокрутка графа
за каждым скроллом кода пробовалась и оказалась хрупкой — гонки по поводу
«какая функция сейчас доминирует на экране», рывки зума). Вместо этого — два
разовых действия:

- **клик по узлу графа** → редактор прыгает на соответствующую строку в
  исходнике и на миг подсвечивает её;
- **курсор в коде → узел в графе**: команда **«Code Graph: показать текущую
  строку в графе»** (по умолчанию `Ctrl+Alt+G`, либо правый клик → пункт в
  контекстном меню редактора) находит узел CFG, покрывающий текущую строку, и
  подсвечивает + прокручивает к нему панель.

Парсинг C — через `web-tree-sitter` (WASM, без нативных модулей), раскладка —
через `@hpcc-js/wasm-graphviz`. Всё считается в extension host; webview только
рендерит готовые SVG и обрабатывает клики/подсветку.

## Как запустить

1. `cd vscode-extension && npm install` (скопирует WASM в `./wasm`).
2. Открыть папку `vscode-extension` в VS Code, нажать **F5** — откроется
   Extension Development Host с загруженной папкой `../CORE`.
3. В нём открыть любой `.c` (например `CORE/user/src/dwin.c`), выполнить команду
   **«Code Graph: открыть панель „Алгоритмы“ рядом с кодом»** (Ctrl+Shift+P).
4. Кликнуть по узлу в графе → редактор прыгнет на нужную строку.
5. Поставить курсор в код, нажать `Ctrl+Alt+G` → граф прокрутится и подсветит
   узел, покрывающий эту строку.

## Архитектура (кратко)

| Файл | Роль |
|------|------|
| `extension.js` | extension host: сборка ленты, команда `codeGraphCfg.locate`, переход по клику |
| `src/cfg-analyzer.mjs` | парсинг + построение CFG всех функций со строками + рендер в SVG |
| `media/main.js` | webview: рендер ленты, клик→navigate, `locate`→подсветка+scrollIntoView |
| `media/main.css` | стили узлов (палитра из темы VS Code через `--vscode-charts-*`) |
| `wasm/` | `tree-sitter.wasm` + `tree-sitter-c.wasm` (кладёт `copy-wasm.js`) |

Это milestone 1. Вне охвата: межфункциональный граф вызовов, обогащение через
clangd (макросы/дефайны), уровень «Связи», расчёты fixed-point/тактов.

## Проверка без VS Code

`node test-analyzer.mjs` — headless-прогон анализатора по `CORE/user/src/dwin.c`:
печатает функции по порядку файла, число узлов CFG и валидность SVG.
